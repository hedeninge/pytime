Just started version3.
Introducing the concept "cup".

"cup" is so that pytime can handle multiple configurations ie "cups".

I wil plobably also change the way systemd-instances are named.
From now where the name is actually the path to the executable 'pytheror',
it will be the name of the "cup".

so "pytime will become the default "cup" and the systemd-instance will be named "pytime".

Then the systemd-service should look for a config-file in known common location.
Which will have to be different depending whether systemd is
running in system-mode as root or user-mode as a normal user.

in user-mode the config-file will be inside the users home-directory.
# /home/sermin/.config/systemd/user/pytime/<cup>.conf
~/.config/pytime/<cupname>.cup.conf

in system-mode the config-file will be in /etc/pytime/<cupname>.cup.conf
